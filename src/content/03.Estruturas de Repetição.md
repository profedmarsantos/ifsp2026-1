# ESTRUTURAS DE REPETIÇÃO EM LINGUAGEM C

## 1. FUNDAMENTAÇÃO TEÓRICA

As estruturas de repetição (laços) permitem executar um bloco de código múltiplas vezes sob controle explícito. Em C, dispomos de três estruturas:

### 1.1 FOR (Controle de Contagem)

**1.1.1 Quando usar:** Quando o número de repetições é conhecido ou previsível.

**1.1.2 Estrutura:**
```c
for (inicialização; condição; atualização) {
    // bloco de comandos
}
```

**1.1.3 Exemplo completo:**
```c
for (int i = 1; i <= 10; i++) {
    printf("%d\n", i);
}
```

**1.1.4 Fluxo lógico:**
1. Inicialização: `int i = 1` (executada uma única vez)
2. Teste da condição: `i <= 10`
3. Se verdadeira: executa o bloco → atualização `i++` → volta ao passo 2
4. Se falsa: encerra o laço

---

### 1.2 WHILE (Pré-teste)

**1.2.1 Quando usar:** Quando zero repetições é um resultado válido ou quando a condição depende de estado externo.

**1.2.2 Estrutura:**
```c
while (condição) {
    // bloco de comandos
    // atualização
}
```

**1.2.3 Exemplo completo:**
```c
int i = 1;  // inicialização
while (i <= 10) {  // condição
    printf("%d\n", i);
    i++;  // atualização
}
```

**1.2.4 Fluxo lógico:**
1. Inicialização: `int i = 1` (antes do laço)
2. Testa a condição: `i <= 10`
3. Se verdadeira: executa o bloco → atualização `i++` → volta ao passo 2
4. Se falsa: encerra o laço

---

### 1.3 DO-WHILE (Pós-teste)

**1.3.1 Quando usar:** Quando a primeira execução é obrigatória antes da verificação.

**1.3.2 Estrutura:**
```c
do {
    // bloco de comandos
    // atualização
} while (condição);
```

**1.3.3 Exemplo completo:**
```c
int i = 1;  // inicialização
do {
    printf("%d\n", i);
    i++;  // atualização
} while (i <= 10);  // condição
```

**1.3.4 Fluxo lógico:**
1. Inicialização: `int i = 1` (antes do laço)
2. Executa o bloco
3. Atualização: `i++`
4. Testa a condição: `i <= 10`
5. Se verdadeira: volta ao passo 2
6. Se falsa: encerra o laço

---

## 2. APLICAÇÃO PRÁTICA: TABUADA COM OS TRÊS MODELOS

### 2.1 Tabuada usando FOR

**2.1.1 Código:**
```c
#include <stdio.h>

int main() {
    int n, i;
    
    printf("Digite um numero: ");
    scanf("%d", &n);
    
    printf("\nTabuada de %d usando FOR:\n", n);
    
    for (i = 1; i <= 10; i++) {
        printf("%d x %d = %d\n", n, i, n * i);
    }
    
    return 0;
}
```

**2.1.2 Análise:**
- Inicialização: `i = 1`
- Condição: `i <= 10`
- Atualização: `i++`
- Número de repetições: exatamente 10

---

### 2.2 Tabuada usando WHILE

**2.2.1 Código:**
```c
#include <stdio.h>

int main() {
    int n, i;
    
    printf("Digite um numero: ");
    scanf("%d", &n);
    
    printf("\nTabuada de %d usando WHILE:\n", n);
    
    i = 1;  // inicialização fora do laço
    
    while (i <= 10) {  // condição
        printf("%d x %d = %d\n", n, i, n * i);
        i++;  // atualização dentro do bloco
    }
    
    return 0;
}
```

**2.2.2 Análise:**
- Inicialização: fora do laço (`i = 1`)
- Condição: `i <= 10`
- Atualização: dentro do bloco (`i++`)
- Mesma funcionalidade do FOR, mas com componentes separados

---

### 2.3 Tabuada usando DO-WHILE

**2.3.1 Código:**
```c
#include <stdio.h>

int main() {
    int n, i;
    
    printf("Digite um numero: ");
    scanf("%d", &n);
    
    printf("\nTabuada de %d usando DO-WHILE:\n", n);
    
    i = 1;  // inicialização fora do laço
    
    do {
        printf("%d x %d = %d\n", n, i, n * i);
        i++;  // atualização dentro do bloco
    } while (i <= 10);  // condição no final
    
    return 0;
}
```

**2.3.2 Análise:**
- Executa pelo menos uma vez (garantia de execução mínima)
- Atualização dentro do bloco
- Condição verificada após execução

## 3. COMPARAÇÃO DIRETA

### 3.1 Tabela Comparativa

| Característica | FOR | WHILE | DO-WHILE |
|----------------|-----|-------|----------|
| Teste da condição | Antes | Antes | Depois |
| Execuções mínimas | 0 | 0 | 1 |
| Ideal para | Contagem previsível | Estado/condição externa | Execução obrigatória |
| Inicialização | No cabeçalho | Antes do laço | Antes do laço |
| Atualização | No cabeçalho | Dentro do bloco | Dentro do bloco |

### 4.2 ORIENTAÇÕES PARA RESOLUÇÃO

**4.2.1 Critérios para escolha da estrutura:**
- Use **FOR** quando souber exatamente quantas vezes precisa repetir
- Use **WHILE** quando a repetição depender de uma condição que pode ser falsa desde o início
- Use **DO-WHILE** quando precisar executar pelo menos uma vez antes de verificar a condição

**4.2.2 Elementos a serem considerados:**
1. Identifique o estado inicial (variáveis de controle)
2. Defina a condição de parada
3. Determine como o estado será atualizado
4. Valide entradas do usuário
5. Previna laços infinitos

## 5. ERROS COMUNS A EVITAR

### 5.1 Laço infinito

**5.1.1 Problema:** Esquecer a atualização da variável de controle

**5.1.2 Exemplo incorreto:**
```c
// ERRADO
while (i <= 10) {
    printf("%d\n", i);
    // faltou i++
}
```

**5.1.3 Exemplo correto:**
```c
// CORRETO
while (i <= 10) {
    printf("%d\n", i);
    i++;
}
```

---

### 5.2 Ponto e vírgula após o cabeçalho

**5.2.1 Problema:** Criar laço vazio infinito

**5.2.2 Exemplo incorreto:**
```c
// ERRADO - cria laço vazio infinito
for (i = 1; i <= 10; i++);
    printf("%d\n", i);
```

**5.2.3 Exemplo correto:**
```c
// CORRETO
for (i = 1; i <= 10; i++)
    printf("%d\n", i);
```

---

### 5.3 Condição de igualdade com incremento

**5.3.1 Problema:** Pode nunca atingir exatamente o valor alvo

**5.3.2 Exemplo perigoso:**
```c
// PERIGOSO - pode nunca atingir exatamente 10
for (i = 1; i != 10; i += 2)
```

**5.3.3 Exemplo seguro:**
```c
// SEGURO
for (i = 1; i <= 10; i += 2)
```

---

### 5.4 Modificar variável de controle dentro do FOR

**5.4.1 Problema:** Quebra a previsibilidade

**5.4.2 Exemplo não recomendado:**
```c
// NÃO RECOMENDADO - quebra a previsibilidade
for (i = 1; i <= 10; i++) {
    printf("%d\n", i);
    i++;  // erro: incrementa duas vezes
}
```

## 6. SOLUÇÕES DOS EXERCÍCIOS

### 6.1 Solução - Exercício 1: Contagem Regressiva

**6.1.1 Estrutura recomendada:** FOR (número conhecido de repetições)

**6.1.2 Código:**
```c
#include <stdio.h>

int main() {
    int n, i;
    
    printf("Digite um numero: ");
    scanf("%d", &n);
    
    // Validação
    if (n < 0) {
        printf("Numero deve ser positivo!\n");
        return 1;
    }
    
    printf("Contagem regressiva: ");
    
    // FOR é ideal pois sabemos exatamente quantas vezes (de n até 0)
    for (i = n; i >= 0; i--) {
        printf("%d ", i);
    }
    
    printf("FIM!\n");
    
    return 0;
}
```

**6.1.3 Justificativa:** FOR é adequado porque conhecemos o número exato de iterações (n+1 vezes, de n até 0).

---

### 6.2 Solução - Exercício 2: Soma de Números

**6.2.1 Estrutura recomendada:** WHILE (número desconhecido de repetições)

**6.2.2 Código:**
```c
#include <stdio.h>

int main() {
    int numero, soma = 0, quantidade = 0;
    
    printf("Digite um numero (0 para sair): ");
    scanf("%d", &numero);
    
    // WHILE é ideal pois não sabemos quantos numeros serao digitados
    while (numero != 0) {
        soma += numero;
        quantidade++;
        
        printf("Digite um numero (0 para sair): ");
        scanf("%d", &numero);
    }
    
    printf("\nTotal de numeros digitados: %d\n", quantidade);
    printf("Soma: %d\n", soma);
    
    return 0;
}
```

**6.2.3 Justificativa:** WHILE é adequado porque não sabemos quantos números serão digitados e zero repetições é um resultado válido (se o usuário digitar 0 imediatamente).

---

### 6.3 Solução - Exercício 3: Validação de Entrada

**6.3.1 Estrutura recomendada:** DO-WHILE (execução obrigatória)

**6.3.2 Código:**
```c
#include <stdio.h>

int main() {
    float nota;
    
    // DO-WHILE é OBRIGATORIO pois precisamos ler pelo menos uma vez
    do {
        printf("Digite uma nota (0-10): ");
        scanf("%f", &nota);
        
        if (nota < 0 || nota > 10) {
            printf("Nota invalida! Tente novamente.\n");
        }
    } while (nota < 0 || nota > 10);
    
    printf("\nNota valida: %.1f\n", nota);
    
    return 0;
}
```

**6.3.3 Justificativa:** DO-WHILE é obrigatório porque precisamos solicitar a entrada PELO MENOS UMA VEZ antes de poder verificar se é válida.

---

### 6.4 Solução - Exercício 4: Tabuada Personalizada

**6.4.1 Estrutura recomendada:** Combinação de DO-WHILE (menu) e FOR (tabuada)

**6.4.2 Código:**
```c
#include <stdio.h>

int main() {
    int base, multiplicador, i;
    int continuar;
    
    // DO-WHILE para o menu principal (executar pelo menos uma vez)
    do {
        printf("Digite o numero base: ");
        scanf("%d", &base);
        
        // Validação
        if (base < 0) {
            printf("Numero deve ser positivo!\n");
            continue;  // volta ao inicio do do-while
        }
        
        printf("Ate qual multiplicador? ");
        scanf("%d", &multiplicador);
        
        // Validação
        if (multiplicador < 1) {
            printf("Multiplicador deve ser pelo menos 1!\n");
            continue;
        }
        
        printf("\nTabuada do %d:\n", base);
        
        // FOR para gerar a tabuada (numero conhecido de repeticoes)
        for (i = 1; i <= multiplicador; i++) {
            printf("%d x %d = %d\n", base, i, base * i);
        }
        
        printf("\nDeseja calcular outra tabuada? (1-Sim / 0-Nao): ");
        scanf("%d", &continuar);
        
    } while (continuar == 1);
    
    printf("Programa encerrado.\n");
    
    return 0;
}
```

**6.4.3 Justificativa:** 
- DO-WHILE para o menu: garante que o programa execute pelo menos uma vez
- FOR para a tabuada: número conhecido de repetições (de 1 até multiplicador)

---

### 6.5 Solução - Exercício 5: Controle de Senha

**6.5.1 Estrutura recomendada:** DO-WHILE (execução obrigatória)

**6.5.2 Código:**
```c
#include <stdio.h>

int main() {
    int senha, senha_correta = 12345;
    
    // DO-WHILE é OBRIGATORIO pois precisamos solicitar a senha pelo menos uma vez
    do {
        printf("Digite a senha: ");
        scanf("%d", &senha);
        
        if (senha != senha_correta) {
            printf("Senha incorreta! Tente novamente.\n");
        }
        
    } while (senha != senha_correta);
    
    printf("Acesso permitido!\n");
    
    return 0;
}
```

**6.5.3 Justificativa:** DO-WHILE é obrigatório porque precisamos solicitar a senha PELO MENOS UMA VEZ antes de poder verificar se está correta. A estrutura garante que o usuário tenha pelo menos uma chance de digitar a senha.
